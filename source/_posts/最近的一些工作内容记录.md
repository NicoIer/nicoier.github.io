时间大概是 2026-06-11 到 2026-06-16。

主要还是 JoltPhysics、BattleProcess、QTE、LOD、技能表现和构建资源。


## 1. 实现萨拉扫描技能

给 `SarahScanCapability` 补大扫描逻辑，参考狮吼技能。监听 `CommonDataTrack` 和 `EventTrack`，在对应帧触发扫描、屏幕变暗和 HUD。

`ScanTrigger` 增加单次显示时长覆盖，不改普通扫描按钮的默认时间。

## 2. 鞭子QTE显示时禁用操作

QTE UI 出现时禁止玩家操作，但不能把 UI 一起隐藏。通过 `CanvasGroup.interactable` 控制交互，阻塞来源保存在 `UseItemQTEUI` 的 `UIVisibilityBlocker` 列表中。

另外处理了 `BattleAttackView` 快进时同步粒子进度，以及服务器 Combo 应用后跳到下一段 Transition 起点。

## 3. QTE目标死亡按失败处理

鞭子 QTE 过程中目标死亡，按失败处理。

顺便整理了 QTE、`Charge`、`Combo`、`TakeBack` 之间的状态切换，重点是把成功、失败和目标死亡后的去向统一。

## 4. OS X键盘移动

Jolt viewer 在 macOS 上鼠标正常，键盘不能移动。最后按 `NSWindow*` 分开保存输入状态，每个 viewer 只读取自己窗口的键鼠输入。

切换窗口时还要清理其他窗口遗留的按键，否则 WASD 会卡在旧窗口。

## 5. Windows字体资源

Windows 上 viewer 找不到字体并报 Fatal Error。

资源改成按平台输出：Windows 放到 `Assets/...`，Mac/Linux 继续用 `Fonts/...` 和 `Shaders/...`。前一次只改 Windows 路径时把 Mac 搞坏了，所以这里必须分开。

## 6. LOD加载和屏占比

角色组件表增加 LOD1、LOD2、LOD3，补了加载、屏占比计算、Inspector 和 Editor 预览。

`SkinnedMeshRenderer` 数据不匹配是因为骨骼路径兜底范围太大，会从外层 root 找到同名节点。现在只从当前 `CharacterViewLOD` 实例根节点解析 prefab path。

## 7. DrawBodyGL旋转

Unity 的 `DrawBodyGL` 和物理调试器旋转不一致，最后查到服务端生成 Box 的逻辑。

`BoxCollider.position` 统一表示几何中心，服务端、编辑器预览和运行时 drawer 全部按同一个定义处理。

## 8. 分离牵手交互按钮

牵手和机关交互拆成两个按钮。移除 `Init()`，改用 `Bind/UnBind`，引用由 prefab 挂载。

`InteractTrigger` 分开保存普通交互和牵手交互状态，两者不再抢同一个 `curInteractable`。

## 9. 输入应用区间

鞭子轻攻击增加输入应用区间。

连击检测区间只记录输入，进入应用区间后才真正切换。点击时间和战斗时间线上的应用时间不能混在一起。

## 10. NetworkReqRsp移除Unity依赖

把 `NetworkReqRsp` 对 `UnityEngine`、`MonoBehaviour`、`UniTask` 和 `Cysharp` 的依赖拆掉，让底层请求响应逻辑可以在非 Unity 环境复用。

## 11. Unity下排除Viewer

`Viewer.cs` 外层增加条件编译，Unity 下不编译 native viewer 相关类型，非 Unity 环境保持原样。

## 12. Mac相机键盘控制

把 viewer 相机控制放到 C# 层。

统一成 `ViewerKeyboardInput`：macOS 用 CoreGraphics 判断焦点和键盘状态，Windows 用 `GetForegroundWindow`、`GetAsyncKeyState`，最后由 `PhysicsViewService` 应用移动。

## 13. Metal library not found

JoltPhysicsApp 在 Mac 上先报 `library not found`，换路径后又报 `library format is not supported`。

原因是 `Shaders.metallib` 太旧。最后从 `joltc` 的 macOS 构建产物同步新文件，同时保留 shader 源码。

## 14. CollideShape

给 `PhysicsMgr` 实现 `CollideShape`，并补 `Body.WorldTransform`、`CenterOfMassTransform`、`InverseCenterOfMassTransform` 和 `JPH_Mat4` 布局测试。

这些测试主要用来检查 PInvoke 层。

## 15. JPH_Mat4结构生成

ClangSharp 把四个 `JPH_Vec4` 生成成了 fixed buffer，改成直接字段：

```csharp
internal partial struct JPH_Mat4
{
    public JPH_Vec4 Column0;
    public JPH_Vec4 Column1;
    public JPH_Vec4 Column2;
    public JPH_Vec4 Column3;
}
```

## 16. CastShape测试

给 `JoltPhysicsSharp` 补 `CastShape` 单元测试，检查命中比例和接触点位置。只验证“不崩”没有意义，关键断言也不能注释掉。

## 17. CollideShape交叠不稳定

`CollideShape_DetectsOverlap` 偶现失败，排查了 `JPH_CollideShapeSettings_Init`、`NarrowPhaseQuery` 和 `CastShape` 命中为 0 的情况。

native settings 结构体需要显式初始化，不能依赖未初始化内存。Vulkan 缺失主要影响 viewer 或 Vulkan 后端，不影响普通物理查询。

## 18. 查询libjoltc.dll导出

用下面的命令读取 Windows DLL 导出表，并保存为 txt 检查 PInvoke：

```shell
llvm-readobj --coff-exports libjoltc.dll
```

## 19. JoltPhysics构建插件复制

Windows 构建后 Plugins 没有复制。现在 item 保存 `Plugins/...` 相对目标路径，构建 target 再根据项目目录和 `OutDir` 解析绝对路径。

## 20. dotnet测试崩溃

Rider 测试退出码 `-1073741819`，命令行也能复现。这个值对应 Windows `0xC0000005 Access Violation`，方向应该放在 native DLL、PInvoke、回调生命周期和 dispose 顺序，不是 Rider。

后续用单项测试、详细日志和 `--blame-crash` 抓 dump。

# 反复出现的问题

- 客户端、服务器和编辑器预览对同一个字段的语义不一致
- native 结构布局、导出函数和资源路径的跨平台差异
- 输入检测、时间线应用和状态切换混在一起
- 测试只跑通，没有断言真正的结果

最近的改动基本都在把这些隐式约定拆开。
